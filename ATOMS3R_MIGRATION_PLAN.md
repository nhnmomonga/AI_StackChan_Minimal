# ATOMS3R AI Chatbot Kit 移植計画

## 概要
このドキュメントは、AI_StackChan_Minimalを ATOM Echo から ATOMS3R AI Chatbot Kit に移植するための詳細な計画です。

## プロジェクト情報
- **現行ハードウェア**: ATOM Echo (ESP32-PICO-D4)
- **ターゲットハードウェア**: ATOMS3R AI Chatbot Kit (ESP32-S3-PICO-1-N8R8)
- **製品URL**: https://www.switch-science.com/products/10487

---

## メインイシュー: ATOM Echo から ATOMS3R AI Chatbot Kit への移植

### 背景
ATOMS3R AI Chatbot Kitは、ATOM Echoと比べて以下の利点があります：
- より強力なプロセッサ (ESP32-S3, デュアルコア Xtensa LX7 @ 240MHz)
- より多くのメモリ (8MB Flash + 8MB PSRAM)
- 内蔵カラーディスプレイ (0.85インチ IPS TFT, 128x128)
- 内蔵センサー (9軸IMU: BMI270 + BMM150)
- 赤外線送信機
- Atomic Echo Base 統合

### 移植の目標
1. 既存の機能をすべて維持
2. 内蔵ディスプレイを活用
3. 追加のセンサーとメモリを将来の拡張に備える
4. ユーザーがどちらのハードウェアでも使用できるようにする（可能な場合）

### イシュー 2: GPIO ピン定義の更新
**優先度**: 高  
**推定工数**: 1-2時間  
**依存**: イシュー 1

#### タスク
- [ ] I2C ピンの変更
  - 現在: GPIO21 (SDA), GPIO25 (SCL)
  - ATOMS3R: GPIO38 (SDA), GPIO39 (SCL) または Grove ポート (G1/G2)
- [ ] LED ピンの確認と更新
  - 現在: GPIO27
  - ATOMS3R: 内蔵 LED ピンの確認が必要
- [ ] ボタンピンの確認
  - ATOMS3R では画面下部にボタンがある
- [ ] オーディオ関連ピンの確認
  - Atomic Echo Base が統合されているため、ピン配置を確認

#### コード変更箇所
- `src/main.cpp` の以下の定義:
```cpp
#define I2C_SDA_PIN 38  // 21 から変更
#define I2C_SCL_PIN 39  // 25 から変更
#define LED_PIN     35  // 確認が必要
```

#### 検証方法
- I2C デバイス（SSD1306）が正しく動作するか確認
- LED が正しく点灯するか確認
- ボタン入力が正しく検出されるか確認

## リスクと対策

### リスク 1: ハードウェア互換性
- **リスク**: M5Unified ライブラリが ATOMS3R を完全サポートしていない可能性
- **対策**: 最新版ライブラリの使用、必要に応じてフォークして修正

### リスク 2: オーディオ品質
- **リスク**: ATOMS3R でのオーディオ設定が ATOM Echo と異なる可能性
- **対策**: 実機での詳細なテストと調整

### リスク 3: メモリ管理
- **リスク**: PSRAM の使用が予期しない問題を引き起こす可能性
- **対策**: 段階的な導入、詳細なメモリプロファイリング

### リスク 4: 後方互換性
- **リスク**: 両方のハードウェアをサポートすることでコードが複雑化
- **対策**: 条件コンパイルの適切な使用、テストの徹底
